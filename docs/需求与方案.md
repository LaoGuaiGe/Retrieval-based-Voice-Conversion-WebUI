## 二、软件本体(A)— 已确认需求

- 产物:自解压**安装 exe**(Inno Setup),免管理员、装用户目录(运行时可写缓存/日志)
- **仅保留实时变声 GUI**,剔除:云端 demo(tools/app.py)、Web 界面(infer-web.py)、公网分享(share=True)、训练模块(infer/modules/train、infer/lib/train)
- GPU 优先 + CPU 回退(PyTorch cu121;16 系显卡自动关半精度)
- 内置模型:
  - 基础(必需):hubert_base.pt、rmvpe.pt
  - 女声 4:雅琳御姐、婉心、诗韵、学姐
  - 男声 2:男神(磁性)、烟嗓(低沉)
- **捆绑 VB-Cable**:安装时默认勾选、单独提权安装(接受一次 UAC)。用户已下载 `E:\rvc-build\VBCABLE_Driver_Pack45.zip`

### UI 重写(已确认)
- **框架:PySide6 彻底重写**(LGPL,适合商业售卖)。放弃现有 FreeSimpleGUI/tkinter 界面
- **配色:纯黑 + 橙色点缀**(电竞风,契合游戏玩家定位)
- **模型选择:方案 A** — 下拉框自动列出 assets/weights 里的模型 + 选中自动配对同名 .index,另留"浏览"按钮手动加载
  - 未来接服务器后,此下拉框演进为"我已购买的模型列表"
- 参数分区 + 中文说明 + 推荐默认值
- 参考风格:je00/Vodka(QML+GLSL,我们 tkinter 做不到 1:1,PySide6 可接近其深色专业质感)

### 已确认的核心修复(与 UI 无关,必做)
1. **中文路径/中文模型名支持**(已实测方案):
   - 根因:`faiss.read_index` 的 C++ FileIOReader 在 Windows 打不开 UTF-8 路径(torch.load 无此问题)
   - 修法:`infer/lib/rtrvc.py` 两处(第 83、200 行)`faiss.read_index(path)` → `faiss.deserialize_index(np.frombuffer(open(path,'rb').read(), dtype=np.uint8))`
   - 新 UI 不再加"拒绝中文"拦截(旧版在 gui_v1.py:668-674)
2. **默认模型修复**:旧 config.json 指向不存在的 kikiV1.pth 导致崩溃。已改 dist 的 config.json 默认指向 nvsheng_yalin_yujie
3. **启动脚本编码**:改用 GBK+CRLF+chcp 936(已改 dist,待重编译进 exe)

## 三、授权系统(B)— 已确认需求

| 决策项 | 选择 |
|--------|------|
| 核心架构 | **搭服务器**(用户已有服务器+域名) |
| 身份 | **邮箱账号**登录 |
| 授权粒度 | **按单个模型授权** |
| 防共享 | **单设备在线互斥**(后登挤掉前者) |
| 联网频率 | **离线宽限期 7 天**(缓存授权,到期再验) |
| 模型分发 | **按需从服务器下载**(装包只带基础模型,音色模型买后再下,走国内镜像加速) |
| 交易 | **虎皮椒**(已开通,微信/支付宝聚合支付) |

### 模型保护方案(已确认现实模型)
- **接受现实**:实时变声要求模型本地内存解密运行,专家可从内存 dump。目标是挡住 95% 普通盗版,非绝对防护
- **加密 + 账号绑定**:
  - 下载到本地的 .pth 是 AES 加密的(直接拷贝转发 = 乱码)
  - 解密密钥由服务器下发,绑定购买账号
  - 客户端只在内存解密,硬盘永远密文
  - 密钥可轮换、可吊销
- **水印(泄露追责)**:
  - ① 容器水印(买家 ID 嵌入加密包元数据,简单)
  - ② 权重水印(给每买家模型权重做极微扰动,唯一可溯源,**需实测验证不劣化音质**)—— 关键溯源手段
- **可选加分**:客户端 Nuitka 编译混淆(抬高内存 dump 门槛)

### 服务器端组件(P1+ 落地)
- 后端服务:账号(注册/登录/邮箱)、授权表(谁买了哪些模型)、单设备互斥 token、模型下发+密钥、虎皮椒支付回调
- 数据库:用户/订单/授权
- 对象存储:加密模型文件
- 用户提供:服务器、域名(HTTPS 证书等进入 P1 再索取具体信息)

## 五、P0 详细计划(PySide6 重写)

- **阶段 0** — 准备:保留 gui_v1.py 作对照;PySide6 版另起新文件(gui_pyside.py);构建环境装 PySide6(国内镜像)
- **阶段 1** — 核心修复:rtrvc.py 的 faiss deserialize;kikiV1、编码修复纳入;冒烟测试
- **阶段 2** — PySide6 主流程(风险最高):窗口骨架+纯黑主题;模型加载/设备选择/开始停止变声;**接线实时音频线程**(GUI↔音频桥接最易翻车,音频内核 rtrvc.py/infer 不动)
- **阶段 3** — 参数区+中文说明+默认值
- **阶段 4** — 纯黑+橙 QSS 美化,中文字体(微软雅黑)
- **阶段 5** — 验证:冒烟测试 → 用户本地真机验证变声效果 → 启动脚本指向新 GUI → 重编译 exe(约 24 分钟,仅验证通过后)

**技术风险提示:** 现有 GUI 与实时音频用 multiprocessing 队列+sounddevice 回调绑定;PySide6 有独立事件循环,需把音频跑独立线程、Qt 信号回传 UI。先跑通"加载+开始/停止"主流程验证桥接,再逐步加。

## 七、关键现实与风险(务必记住)

1. **模型不可能绝对防提取**:实时变声=本地内存解密,专家能 dump。加密+账号绑定挡 95% 普通盗版,水印用于追责。护城河靠**持续服务**(上新音色/定制训练/技术支持),非纯技术
2. **授权系统是全栈工程**:客户端改造+后端+数据库+对象存储+支付+运维,远大于变声器本体
3. **PyQt 重写风险**:GUI↔实时音频桥接最易引入 bug,音频内核不动可降险
4. **虎皮椒**:开户费 118 + 平台费 1~2% + 官方 0.6%;注意 2025-11-6 起微信跳转/小程序支付临时关闭(PC 扫码不受影响),需自行核实最新状态
5. **音质/延迟无法由 AI 验证**:构建机 4GB 显存无音频回路,只能验证"能加载初始化",实际效果需用户真机测
